PLAYER LOOP

Observe → Interpretation → Choice → State → Echo → Ending

玩家读到的不是分支数量，而是“有限信息中的行动会被世界记住”。

Observe展开观察，获得细节和线索。
→
Interpretation玩家重新理解当前局面。
→
Choice底部行动栏出现可承担的选择。
→
State写入 flags、clues、relationship。
→
Echo后续场景、复盘、结局给出回声。
INTERACTION MAP

隐藏不是陷阱，隐藏是理解后的可见行动

Observe 负责改变理解和解锁行动；Choice 负责推进剧情并承担代价。

Scene Text 正文承担氛围、事实和可观察对象。
  • 锁屏短信
  • 远程清除提示
  • 订单与设备痕迹
Observe Chain 同一时刻只展开一条观察链。
  • 顶层 observe
  • 嵌套 observe
  • 重复查看不重复结算
Hidden State 状态隐性存在，但必须有文本暗示。
  • clues / flags
  • trust / suspicion
  • ending_tags
Visible Choice 关键行动必须在底部行动栏可见。
  • 证据足够才出现
  • 选项完整可见
  • 复盘解释未解锁原因
RUNTIME ARCHITECTURE

Kernel → Loop → Content → Runtime

四层单向依赖：契约稳定，玩法加严，作品装包，运行时按 Kernel 主版本解释。

L1 · Kernelstate、surface、action、progression、save、verification 封闭契约。
L2 · Loop调查规则、schema extension、G3–G5 与体验阈值。
L3 · Contentpack.json、game、state registry、资产与 provenance。
L4 · Runtime包加载、通用引擎、存档、surface renderer 与玩法 UI adapter。
AI GENERATION PIPELINE

Agent 用摘要绑定的人工节点控制生成与发布

LLM 只处理小型创意 artifact；ID、结构、组装、repair 和门禁保持确定性。

1Brief Artifact锁定题材、边界、玩法包和 Experimental opt-in。
2Brief Receipt摘要过期或未批准时立即停顿。
3State + Blueprint确定性规划状态、场景预算与结局覆盖。
4Blueprint Receipt未批准不批量生成 scene artifacts。
8Release Bundle重算 G1–G5 后只构建选定内容包，不部署。
7Release Receipt与 release candidate 摘要绑定。
6G1–G5Kernel、玩法、BFS witness 与浏览器重放全绿。
5Pack + Provenance组装 V1 内容包、脱敏 trace 与 V0 投影。
RELEASE GATES

G1–G5 工程闭环，G6 真人证据清偿债务

所有门禁共享 GateResult；自动全绿允许打包，但发布仍需绑定摘要的人工批准。

G1manifest、Kernel、玩法扩展、资产与 provenance。
G2引用、状态读写类型、公平性与结构可达。
G3调查玩法预算、层级、明显 anchor 与解锁契约。
G4有限状态 BFS 自动产出所有结局 witness。
G5浏览器重放、存档、fallback、标记与无障碍。
G6 · Debt5–8 人阈值与归因凭证；未达标降级 Experimental。
