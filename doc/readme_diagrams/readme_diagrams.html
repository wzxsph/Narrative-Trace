PLAYER LOOP

Observe → Interpretation → Choice → State → Echo → Ending

玩家读到的不是分支数量，而是“有限信息中的行动会被世界记住”。

Observe展开观察，获得细节和线索。
→
Interpretation玩家重新理解当前局面。
→
Choice底部行动栏出现可承担的选择。
→
State写入 flags、clues、relationship。
→
Echo后续场景、复盘、结局给出回声。
INTERACTION MAP

隐藏不是陷阱，隐藏是理解后的可见行动

Observe 负责改变理解和解锁行动；Choice 负责推进剧情并承担代价。

Scene Text 正文承担氛围、事实和可观察对象。
  • 锁屏短信
  • 远程清除提示
  • 订单与设备痕迹
Observe Chain 同一时刻只展开一条观察链。
  • 顶层 observe
  • 嵌套 observe
  • 重复查看不重复结算
Hidden State 状态隐性存在，但必须有文本暗示。
  • clues / flags
  • trust / suspicion
  • ending_tags
Visible Choice 关键行动必须在底部行动栏可见。
  • 证据足够才出现
  • 选项完整可见
  • 复盘解释未解锁原因
RUNTIME ARCHITECTURE

从结构化内容到纸墨竖屏体验

运行时不临场编剧情，而是稳定解释 game.json、状态、存档和 UI。

Brief主题、角色、章节目标、节点预算。
ValidationJSON schema、结构校验、内容 QA。
Runtime确定性状态机、observe 解锁、choice 跳转。
SavelocalStorage 存档、迁移、坏档恢复。
UI纸墨风格、正文滚动、选项固定可见。
AI GENERATION PIPELINE

Agent 生成的是可审查 artifact，不是不可控正文洪流

每一步都产出结构化文件，并经过校验与人工可读的 trace。

1Load Brief读取题材、主题问题和创作约束。
2Plan Story规划章节问题、主路径和结局覆盖。
3State Schema设计 flags、clues、relationships。
4Scene Blueprint锁定场景预算、入口和关键行动。
8Write Trace保存每步输入、输出和验证结果。
7Export Game导出可运行 game.json 和报告。
6Validate / Repair校验结构、内容 QA、必要时修复。
5Scene Artifacts生成 observe、choice、effects 草案。
RELEASE GATES

从“能生成”到“能交付”的闸门

Demo 不能只看跑通，还要看是否公平、可达、可复盘。

Schema所有字段声明完整，无未声明变量。
Structure无孤儿节点、无死 choice、结局可达。
Content QA重大状态变化有文本暗示，选择不是伪影响。
Browser E2E三条主结局、复盘、存档恢复可跑通。
Experience Risk隐藏选项不得制造不公平或黑箱惩罚。
